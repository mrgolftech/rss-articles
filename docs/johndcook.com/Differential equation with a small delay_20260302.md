{'type': 'text/html', 'language': None, 'base': 'https://www.johndcook.com/blog/feed/', 'value': '<p>In grad school I specialized in differential equations, but never worked with delay-differential equations, equations specifying that a solution depends not only on its derivatives but also on the state of the function at a previous time. The first time I worked with a delay-differential equation would come a couple decades later when I did some modeling work for a pharmaceutical company.</p>\n<p>Large delays can change the qualitative behavior of a differential equation, but it seems plausible that sufficiently delays should not. This is correct, and we will show just how small &#8220;sufficiently small&#8221; is in a simple special case. We&#8217;ll look at the equation</p>\n<p style="padding-left: 40px;"><em>x</em>′(<em>t</em>) = <em>a x</em>(<em>t</em>) + <em>b x</em>(<em>t</em> − τ)</p>\n<p>where the coefficients <em>a</em>\xa0and <em>b</em> are non-zero real constants and the delay τ is a positive constant. Then [1] proves that the equation above has the same qualitative behavior as the same equation with the delay removed, i.e. with τ = 0, provided τ is small enough. Here &#8220;small enough&#8221; means</p>\n<p style="padding-left: 40px;">−1/<em>e</em> &lt;\xa0<em>bτ</em> exp(−<em>a</em>τ) &lt;\xa0<em>e</em></p>\n<p>and</p>\n<p style="padding-left: 40px;"><em>a</em>τ &lt; 1.</p>\n<p>There is a further hypothesis for the theorem cited above, a technical condition that holds on a nowhere dense set. The solution to a first order delay-differential like the one we&#8217;re looking at here is not determined by an initial condition\xa0<em>x</em>(0) =\xa0<em>x</em><sub>0</sub> alone. We have to specify the solution over the interval [−τ, 0]. This can be any function of <em>t</em>, subject only to a technical condition that holds on a nowhere-dense set of initial conditions. See [1] for details.</p>\n<h2>Example</h2>\n<p>Let&#8217;s look at a specific example,</p>\n<p style="padding-left: 40px;"><em>x</em>′(<em>t</em>) = −3<em>\xa0x</em>(<em>t</em>) + 2<em>\xa0x</em>(<em>t</em> − τ)</p>\n<p>with the initial condition <em>x</em>(1) = 1. If there were no delay term τ, the solution would be\xa0<em>x</em>(<em>t</em>) = exp(1 −\xa0<em>t</em>). In this case the solution monotonically decays to zero.</p>\n<p>The theorem above says we should expect the same behavior as long as</p>\n<p style="padding-left: 40px;">−1/<em>e</em> &lt;\xa02τ exp(3τ) &lt; <em>e</em></p>\n<p>which holds as long as τ &lt; 0.404218.</p>\n<p>Let&#8217;s solve our equation for the case τ = 0.4 using Mathematica.</p>\n<pre>tau = 0.4\nsolution = NDSolveValue[\n    {x\'[t] == -3 x[t] + 2 x[t - tau], x[t /; t &lt;= 1] == t }, \n    x, {t, 0, 10}]\nPlot[solution[t], {t, 0, 10}, PlotRange -&gt; All]</pre>\n<p>This produces the following plot.</p>\n<p><img class="aligncenter size-medium" height="294" src="https://www.johndcook.com/delaydiff1.png" width="480" /></p>\n<p>The solution initially ramps up to 1, because that&#8217;s what we specified, but it seems that eventually the solution monotonically decays to 0, just as when τ = 0.</p>\n<p>When we change the delay to τ = 3 and rerun the code we get oscillations.</p>\n<p><img class="aligncenter size-medium" height="294" src="https://www.johndcook.com/delaydiff2.png" width="480" /></p>\n<p>[1] R.
